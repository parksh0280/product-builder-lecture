Git 마스터: Rebase와 Cherry-pick으로 완성하는 깔끔한 커밋 히스토리

2026. 02. 18
Admin

커밋 메시지가 'fixed bug', 'update'로 가득 차 있지는 않나요? Git 히스토리는 프로젝트의 족보와 같습니다. 동료 개발자들이 이해하기 쉽고, 추적이 용이한 깔끔한 히스토리를 만드는 고급 기술들을 알아봅니다.

1. Interactive Rebase (`git rebase -i`)

커밋을 합치고(squash), 이름을 바꾸고(reword), 순서를 바꾸는 마법 같은 도구입니다. 원격 저장소에 푸시하기 전, 지저분한 작업 로그를 단 하나의 의미 있는 커밋으로 정리하세요.

2. Cherry-pick: 필요한 것만 골라담기

다른 브랜치의 특정 기능이나 버그 수정 내용만 현재 브랜치로 가져오고 싶을 때 사용합니다. 불필요한 머지 없이 코드의 정수만을 추출할 수 있습니다.

3. Reflog: Git의 타임머신

실수로 중요한 커밋을 삭제하거나 리베이스 도중 코드가 꼬였다면? `git reflog`는 당신이 수행한 모든 작업의 이력을 보여줍니다. '삭제된 커밋'도 복구할 수 있는 최후의 수단입니다.

협업의 기본은 커뮤니케이션입니다

깔끔한 Git 히스토리는 문서 작성만큼이나 중요한 커뮤니케이션 도구입니다. 미래의 당신과 동료들을 위해 오늘부터 커밋 하나에도 정성을 들여보세요.
