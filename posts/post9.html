"코드는 컴퓨터가 이해하기 위해서가 아니라, 사람이 이해하기 위해 작성하는 것이다." 켄트 벡의 이 오래된 격언은 AI가 코드를 생성하는 2026년에도 여전히 유효합니다. 아니, 오히려 AI가 쏟아내는 수많은 코드 더미 속에서 진주를 가려내고 유지보수 가능한 상태로 관리해야 하는 현재, 클린 코드(Clean Code)의 가치는 그 어느 때보다 빛을 발합니다. 오늘은 자바스크립트와 타입스크립트 환경에서 코드의 가독성, 유지보수성, 그리고 확장성을 극대화하기 위한 현대적인 클린 코드 기법들을 정리해 봅니다.

1. 의미 있는 이름 짓기: 변수명은 설명서다

`data`, `info`, `temp` 같은 이름은 코드의 의도를 숨깁니다. 훌륭한 변수명은 그 자체로 주석의 역할을 해야 합니다. 2026년의 개발자들은 의도를 분명히 드러내는 이름을 짓기 위해 고군분투합니다.

  • 나쁜 예: `const d = new Date();`
  • 좋은 예: `const currentTimestamp = new Date();`
  • 함수는 동사로: `getUserData` 보다는 `fetchUserProfile`이 더 구체적인 동작을 설명합니다.

2. 함수는 오직 한 가지 일만 해야 한다 (SRP)

하나의 함수가 데이터를 가져오고, 가공하고, 화면에 그리기까지 한다면 그 함수는 '괴물'이 됩니다. 함수를 작게 쪼개고 각각의 함수가 단 하나의 책임(Single Responsibility)만 지게 하세요. 이는 테스트를 쉽게 만들고, 코드 재사용성을 높여줍니다. 만약 함수 이름에 `And`나 `Or`가 들어간다면, 그 함수는 이미 너무 많은 일을 하고 있다는 신호입니다.

3. 부정문보다는 긍정문을 사용하라

`if (!isNotValid)`와 같은 이중 부정은 뇌의 처리 속도를 늦춥니다. 가능한 한 긍정문(`if (isValid)`)을 사용하여 로직을 단순화하세요. 또한, 복잡한 조건문은 변수나 함수로 추출하여 이름을 붙여주는 것이 좋습니다. "왜 이 조건이 필요한가"를 코드가 직접 설명하게 하세요.

4. Early Return 패턴의 활용

깊게 중첩된 `if-else` 문은 '장풍 코드'를 만듭니다. 예외 상황이나 조건이 맞지 않는 경우를 함수 시작 부분에서 먼저 처리하고 반환(return)하세요. 이렇게 하면 함수의 주요 로직이 들여쓰기 없이 깔끔하게 유지되어 흐름을 파악하기 훨씬 쉬워집니다.

Before & After 예시

Bad Practice

function p(u) {
  if (u !== null) {
    if (u.a > 18) {
      send(u);
    } else {
      alert('too young');
    }
  }
}

Clean Code

function processUser(user) {
  if (!user) return;
  
  const isAdult = user.age > 18;
  if (!isAdult) {
    alert('미성년자는 접근할 수 없습니다.');
    return;
  }
  
  sendNotification(user);
}

5. 매직 넘버(Magic Number)를 상수로 치환하라

코드 중간에 튀어나오는 `86400` 같은 숫자는 무엇을 의미하는지 알기 어렵습니다. `SECONDS_IN_A_DAY`와 같이 명확한 이름을 가진 상수로 정의하세요. 이는 코드의 가독성을 높일 뿐만 아니라, 나중에 값이 변경될 때 단 한 곳만 수정하면 되는 유지보수의 이점을 제공합니다.

마치며: 클린 코드는 배려다

클린 코드는 단순히 보기 좋은 코드를 짜는 기술이 아닙니다. 그것은 미래의 나, 그리고 함께 일하는 동료들에 대한 '최고의 배려'입니다. 오늘 여러분이 무심코 남긴 '돌아가기만 하는 코드'는 내일 누군가의 밤샘 작업 원인이 될 수 있습니다. 매 커밋마다 조금씩 더 깨끗한 코드를 남기려 노력해 보세요. 보이스카우트 규칙처럼, 캠프장을 떠날 때 처음 왔을 때보다 더 깨끗하게 만드는 마음가짐이 훌륭한 엔지니어를 만듭니다.
